(Optional but Recommended) Using a Custom Config for Easier Updates

To minimize inconvenience during theme updates, it’s a good practice to create a separate configuration file for the theme. Follow these steps:

  1. In the root directory of your Hexo project, create a new file named _config.butterfly.yml.
  2. Copy the content from the theme’s _config.yml file located in your-project-name/themes/butterfly/config.yml and paste it into the newly created _config.butterfly.yml.

Important Notes:

  • Copy the theme’s _config.yml, not Hexo’s default _config.yml.
  • Do not delete the theme’s _config.yml.
  • From now on, you will only need to configure your theme through _config.butterfly.yml. We will not change themes/butterfly/_config.yml anymore. And _config.yml mentioned next in this guide is your-project-name/_config.yml.

Hexo automatically merges the configurations from both _config.yml and _config.butterfly.yml. If there are conflicting settings, the values from _config.butterfly.yml will take precedence.
